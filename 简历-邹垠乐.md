## 项目经验

:::left
### SwiftStock电商仓库管理系统
:::right
**2024.09 - 至今**
:::

`Spring Boot` `MySQL` `MyBatis` `Spring Security` `Vue.js` `Element Plus`

- **项目描述**：基于Spring Boot + Vue的电商仓库管理系统，采用前后端分离架构设计。系统实现了商品管理、订单处理、库存控制、数据统计等核心功能，支持多级分类管理、库存预警机制、订单状态流转校验等业务逻辑，为电商仓库日常运营提供信息化支持。
- **工作内容**：
  - 负责后端核心功能开发，采用MVC分层架构设计并实现30+个RESTful API接口，使用Spring Security实现基于角色的权限控制
  - 设计并实现6张核心数据库表（admin、category、product、orders、order_item、inventory_record），使用MyBatis实现数据持久化，通过MySQL触发器自动生成商品编码（P00001格式）和订单编号（ORD202501060001格式）
- **技术亮点**：
  - 使用`@Transactional`注解和订单状态枚举（OrderStatus）保障订单创建与库存扣减的事务一致性，通过`canTransitionTo()`方法实现8种订单状态的合法流转校验（待付款→已付款→配货中→已发货→已送达→已完成/取消/退款）
- **技术难点**：
  - 订单创建涉及订单表、订单项表和库存扣减的多表事务操作，使用`@Transactional`保障ACID特性；当库存不足时通过异常机制触发事务回滚，有效防止超卖现象

### 宠物管理系统
:::right
**2024.06 - 2024.09**
:::

`Spring Boot` `MySQL` `MyBatis` `Spring Security` `JWT`

- **项目描述**：基于Spring Boot的宠物管理系统后端服务，采用前后端分离架构设计。系统实现了用户身份认证、宠物信息管理、健康记录管理等核心功能，支持多条件组合查询和数据统计分析，为宠物店日常管理提供信息化支持。
- **工作内容**：
  - 负责后端核心功能开发，采用MVC分层架构设计并实现15+个RESTful API接口，使用Spring Security + JWT实现无状态身份认证和授权机制
  - 设计并实现3张数据库表（user、pet、health_record），使用MyBatis实现数据持久化，通过动态SQL和LEFT JOIN优化多表关联查询性能
- **技术亮点**：
  - 使用MyBatis动态SQL（`<if>`、`<where>`标签）和LEFT JOIN实现跨表多维度条件搜索（按宠物名称、主人姓名、健康状态等），查询响应时间控制在200ms以内
- **技术难点**：
  - 健康记录查询需同时展示宠物信息和主人信息，涉及3张表关联；通过LEFT JOIN一次性查询避免N+1查询问题，显著提升查询性能
